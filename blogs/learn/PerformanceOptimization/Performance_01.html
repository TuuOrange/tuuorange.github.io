    I Know 性能优化（一）

    TuuOrange 3/10/2025 工作工作总结学习学习总结性能优化Webpack

    此系列为我遇到过的一些关于优化的问题，不定期更新，是个工作与学习进步记录 🩵 ～

    # 当我使用 buddleAnalyzerPlugin/vue-cli-service build --report 查看打包文件体积

    查看打包文件体积，会发现有一些第三方库的文件体积很大，影响了整个项目的加载速度。

    这时候我们可以使用webpack的externals来解决。

    # why we use externals?

    因为externals能够将我们声明的第三方依赖库，转为，CDN 外部引入。这样第三方库就不会被打包啦～

    默认情况下，VUE 的开发模式与生产模式，是共用的同一个打包的入口文件（即 src/main.js）。为了将项目的开发过程和发布过程分离，我们可以为两种模式，各自指定打包的入口文件。

    ① 开发模式的入口文件为 src/main-dev.js

    ② 发布模式的入口文件为 src/main-prod.js

    # how we do?

    按需创建 vue.config.js，在 vue.config.js 中配置

    module.exports = {
  chainWebpack: (config) => {
    // 发布模式
    config.when(process.env.NODE_ENV === "production", (config) => {
      config.entry("app").clear().add("./src/main-prod.js");

      config.set("externals", {
        vue: "Vue",
        "vue-router": "VueRouter",
        axios: "axios",
        lodash: "_",
        echarts: "echarts",
        nprogress: "NProgress",
        "vue-quill-editor": "VueQuillEditor",
      });
    });
    // 开发模式
    config.when(process.env.NODE_ENV === "development", (config) => {
      config.entry("app").clear().add("./src/main-dev.js");
    });
  },
};

    # 拓展

    不同的打包内容，也可以设置不同的标题。 主要实现：在 public/index.html 中，可以根据isProd的值（通过 HtmlWebpackPlugin 中的 options 来添加定义），来决定如何渲染页面结构。

    /*vue.config.js;*/
config.plugin("html").tap((args) => {
  args[0].isProd = true;
  return args;
}); // 在生产模式下，开发模式为false

    /*html中*/
<!-- 按需渲染页面的标题 -->
<title><%= htmlWebpackPlugin.options.isProd ? '' : 'dev - ' %>电商后台管理系统</title>

<!-- 按需加载外部的 CDN 资源 -->
<% if(htmlWebpackPlugin.options.isProd) { %>
<!-- 通过 externals 加载的外部 CDN 资源-->
<% } %>

    # 路由懒加载（按需加载）

    也许，你觉得路由懒加载很简单，不就是用 import 动态引入路由组件然后 vueRouter 自动处理嘛？是，但是你知道有哪些要注意的事项吗？👀

    # webpack 自动识别 import()语法并分割代码，需要保证 output.chunkFilename 配置正确

    output.chunkFilename 用来定义非入口 chunk 文件的命名规则.

    module.exports = {
  output: {
    filename: "[name].bundle.js", // 入口文件命名规则
    chunkFilename: "[name].[contenthash].chunk.js", // 非入口 chunk 命名规则
    path: path.resolve(__dirname, "dist"),
  },
};

    默认情况下，动态导入的 chunk 名称是自动生成的 id（如 1.js），若要[name]占位符有效，可以通过魔法注释显式命名：

    // Vue/React 动态导入组件时指定名称
const Home = () => import(/* webpackChunkName: "home" */ "./Home.vue");

    为了优化缓存，生产环境应该在文件名中添加 hash（指纹策略）：

    chunkFilename: process.env.NODE_ENV === 'production'
  ? '[name].[contenthash:8].chunk.js'
  : '[name].chunk.js',

    # 如果浏览器加载 404

    原因：publicPath 配置错误或者是 chunk 文件未生成正确的路径

    # 解决：

    1. 检查 output.publicPath 和 output.path
    2. 执行构建后，查看 dist/目录下是否生成 chunk 文件（根据项目部署路径调整）
    const path = require('path');
const { CleanWebpackPlugin } = require('clean-webpack-plugin');

module.exports = {
  mode: 'production',
  entry: './src/index.js',
  output: {
    filename: '[name].[contenthash:8].bundle.js',
    chunkFilename: '[name].[contenthash:8].chunk.js',
    path: path.resolve(__dirname, 'dist'),
    publicPath: '/', // 根据项目部署路径调整
  },
  optimization: {
    splitChunks: {
      chunks: 'all',\n name: (module, chunks, cacheGroupKey) => {
      // 自定义拆分后的 chunk 名称（否则默认是数字 ID）
      return `${cacheGroupKey}-${chunks.map(chunk => chunk.name).join('~')}`;
    },
    },
  },
  plugins: [new CleanWebpackPlugin()],
};

    1. 安装开发依赖@babel/plugin-syntax-dynamic-import包（旧项目兼容）
    2. 在 babel.config.js 配置文件声明该插件

    [
      "component",
      {
        libraryName: "element-ui",
        styleLibraryName: "theme-chalk",
      },
    ],
    // 发布产品时候的插件数组
    [...prodPlugins],
    ["@babel/plugin-syntax-dynamic-import"],
  ],

    1. 将路由按需加载

    # Gzip 压缩

    谈到性能优化，首先想到的都是缓存压缩之类的，那 Gzip 具体是怎么实现的呢？

    Gzip 压缩其实主要需要通过服务端配置 Nginx 支持 Gzip 压缩，服务端首先考虑加载预压缩.gz 文件

    响应头包含 Content-Encoding: gzip 说明 Gzip 生效

    前端 - 可以通过CompressionPlugin插件对包进行 gzip/brotli 压缩

    const CompressionPlugin = require("compression-webpack-plugin");
module.exports = {
  plugins: [
    new CompressionPlugin({
      filename: "[path][base].gz", // 生成文件名格式
      algorithm: "gzip", // 压缩算法
      test: /\.(js|css|html|svg)$/, // 匹配文件类型
      threshold: 10240, // 只压缩大于 10KB 的文件
      minRatio: 0.8, // 压缩率低于 0.8 时才压缩
    }),
  ],
};

    Brotli 压缩比 gzip 压缩更高效

    参考自：https://juejin.cn/post/7030386682868269064#heading-8
