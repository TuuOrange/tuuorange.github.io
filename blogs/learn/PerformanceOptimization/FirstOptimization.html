    TuuOrange 4/13/2023 性能优化首屏优化

    性能优化的目的都是为了提升用户体验，以用户体验为先。

    # 首屏加载优化

    要优化首屏加载的速度，首先需要确认下首屏加载时的常见指标，针对这些指标进行优化。

    # 指标

    • FCP：白屏时间，即文本绘制的时间

    • Speed Index：首屏时间

    • TTI：第一次可交互的时间

    在选择优化方案之前，我们先要确定有哪些部分影响了我们首屏加载的速度，进行针对性的优化，可以通过一些调试工具获取我们所需要的指标。比如说 network、lighthouse、dist 目录....

    # 静态资源

    将静态资源存储到 webpack 不处理的文件中，存放到 assets 文件中。

    使用 image-webpack-loader 将图片在构建过程中压缩图片。 也可以自己进行压缩之后再放入，可以使用 tinyPng 网站。 如果需要加载一张效果图的话，可以将图片分割，用 CSS 布局将图片拼接到一起。

    # 使用 webP 图片

    webP 图片质量好体积小！

    手动转化成 webP 图片：webP-converter 工具、智图（其二选）

    自动转化成 webP 图片：image-min-webP 等插件

    # 优化 SVG 图标

    使用 iconfont 引入图标，不能够按需加载，而且操作麻烦，更换图标和上传自定义的图标都很不方便。

    所以，可以将 svg 图标放入 src/icons/svg 目录下，在全局引入 svg 组件（svg-sprite-loader），使用时复制文件名即可。

    // install
npm i svg-sprite-loader -D
// vue.config.js
chainWebpack: (config) => {
    // SVG处理
    config.module
      .rule('svg')
      .exclude.add(resolve('src/icons/svg'))
      .end()
    config.module
      .rule('icons')
      .test(/\.svg$/)
      .include.add(resolve('src/icons/svg'))
      .end()
      .use('svg-sprite-loader')
      .loader('svg-sprite-loader')
      .options({
        symbolId: 'icon-[name]'
      })
      .end()
}

// src/icons/index.js
const req = require.context('./svg', false, /\.svg$/)
const requireAll = requireContext => requireContext.keys().map(requireContext)
requireAll(req)

// main.js
import '@/icons'

// src/icons/index.js
import Vue from 'vue'
import CaSVG from '@/components/ca-svg'
Vue.component('ca-svg', CaSVG)

// src/components/ca-svg.vue
<template>
  <svg :class="svgClass" aria-hidden="true" v-on="$listeners" :style="svgStyle">
    <use :xlink:href="iconName" />
  </svg>
</template>

    # 优化 moment、moment-timezone 体积

    # 不打包 moment 文件

    使用内置的 ignorePlugin。

    // webpack plugins
plugins: [
  // Ignore all locale files of moment.js
  new webpack.IgnorePlugin(/^\.\/locale$/, /moment$/),
],

    # 移除 moment-timezone

    moment-timezone 中包含 moment，所以不需要额外引入 moment-timezone。

    # 使用 dayjs 替换 moment

    dayjs 能够实现大多数 moment 的功能，而且体积很小。

    可以使用 antd-dayjs-webpack-plugin 将 moment 替换成 dayjs。

    // vue.config.js
const AntdDayjsWebpackPlugin = require('antd-dayjs-webpack-plugin')
configureWebpack: {
    ...
    plugins: [
      new AntdDayjsWebpackPlugin({ preset: 'antdv3', replaceMoment: true })
    ],
}

    # 优化 core-js 体积

    core-js 是浏览器新 API 的 polypill，主要是为了项目的兼容，所以不会用到的地方可以移除掉。

    # 调整.browserslistrc

    指定需要兼容的版本，少做兼容工作。

    # 调整 useBuiltIns

    将 useBuiltIns 值转为 usage，按需引入。

    // babel.config.js
module.exports = {
  presets: [
    "@vue/cli-plugin-babel/preset",
    [
      "@babel/preset-env",
      {
        useBuiltIns: "usage", // entry，usage
        corejs: 3,
      },
    ],
  ],
  plugins,
};

    # HTTP2

    HTTP2 是 HTTP 协议的第二个版本，相较于 HTTP1 速度更快、延迟更低、功能更多，只是兼容性比 HTTP1 少一些。使用 HTTP2，可以在一个 TCP 连接分帧处理多个请求，也就是多路复用。

    # Nginx 配置

    // nginx.conf
listen 443 http2;

nginx -s stop && nginx

    # Gzip 压缩传输

    使用 Gzip 压缩可以将文本文件减少 2/3 的体积。响应头 Content-Encoding 指定 Gzip 后，浏览器就会对其进行解压。

    # Nginx 配置 Gzip

    #开启和关闭gzip模式
gzip on;
#gizp压缩起点，文件大于1k才进行压缩
gzip_min_length 1k;
# gzip 压缩级别，1-9，数字越大压缩的越好，也越占用CPU时间
gzip_comp_level 6;
# 进行压缩的文件类型。
gzip_types text/plain application/javascript application/x-javascript text/css application/xml text/javascript ;
# nginx对于静态文件的处理模块，开启后会寻找以.gz结尾的文件，直接返回，不会占用cpu进行压缩，如果找不到则不进行压缩
gzip_static on
# 是否在http header中添加Vary: Accept-Encoding，建议开启
gzip_vary on;
# 设置gzip压缩针对的HTTP协议版本
gzip_http_version 1.1;

    但是利用 Nginx 配置 Gzip 的话，会使 Nginx 开销变大，可以直接打包压缩 Gzip 文件，由服务器直接返回。（但是图片最好不要用 Gzip 压缩的，需要另外处理）

    // vue.config.js
const CompressionPlugin = require("compression-webpack-plugin");
// gzip压缩处理
chainWebpack: (config) => {
  if (isProd) {
    config.plugin("compression-webpack-plugin").use(
      new CompressionPlugin({
        test: /\.js$|\.html$|\.css$/, // 匹配文件名
        threshold: 10240, // 对超过10k的数据压缩
        deleteOriginalAssets: false, // 不删除源文件
      })
    );
  }
};

    # 预处理

    预处理就是预先处理请求加载比较慢的资源，先提示浏览器所要加载请求的资源，然后浏览器可以提前获取资源或者是将资源进行缓存，在客户端体验上，会加速首屏加载的时间。

    vue 脚手架 3 默认会使用 preload-webpack-plugin 对 chunk 资源做 preload、prefetch 处理，入口文件 preload 预加载，路由 chunk 则是 prefetch 预请求。

    # OSS 托管+CDN 加速

    OSS 静态资源存储，不仅安全稳定，还支持压缩、水印、格式转换等，OSS 本身传输已经加速，但是加上 CDN 的话，提升更大。

    CDN 加速原理是把提供的域名作为源站，将源内容缓存到边缘节点。当客户读取数据时，会从最适合的节点（一般来说就近获取）获取缓存文件，以提升下载速度。
