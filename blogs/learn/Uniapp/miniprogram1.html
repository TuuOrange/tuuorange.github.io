    初步了解微信小程序开发-基础篇

    vuePress-theme-reco TuuOrange    2025

    初步了解微信小程序开发-基础篇

    TuuOrange 4/13/2023 微信小程序uni-app

    # 微信小程序

    # 小程序与普通网页的不同

    1、运行环境不同 - 网页运行在浏览器环境之中 小程序运行在微信环境之中

    2、API 不同 - 由于运行环境的差异，小程序中无法调用 BOM 和 DOM 的 API，小程序有微信环境提供的各种 API，例如：地理位置、扫码支付之类的

    3、开发模式不同 - 网页开发模式是浏览器+代码编辑器，而小程序有自己的开发模式

    # 项目结构

    pages 用来存放所有小程序的页面

    utils 用来存放工具性质的模块

    app.js 是小程序项目的入口文件

    app.json 是小程序的全局配置文件

    app.wxss 是小程序的全局样式文件

    project.config.json 是小程序的配置文件

    sitemap.json 用来配置小程序及其页面是否允许被微信索引

    # 小程序页面的组成部分

    .js 文件（页面的脚本文件，存放页面的数据、事件处理函数等）

    .json 文件（当前页面的配置文件，配置窗口的外观、表现等）

    .wxml 文件（页面的模板结构文件）=== HTML

    .wxss 文件（当前页面的样式表文件） === CSS

    # json 配置文件

    1、 项目根目录中的 app.json 配置文件

    2、 项目根目录中的 project.config.json 配置文件

    3、 项目根目录中的 sitemap.json 配置文件

    4、 每个页面中的.json 配置文件

    # app.json

    当前小程序的全局配置，包含小程序的所有页面路径、窗口外观、界面表现、底部 tab 等。

    {
  // 用来记录当前小程序所有页面的路径
  "pages": [
    "pages/index/index",
    "pages/logs/logs"
  ],
  // 全局定义小程序所有页面的外观配置
  "window": {
    "backgroundTextStyle": "light",
    "navigationBarBackgroundColor": "#fff",
    "navigationBarTitleText": "Weixin",
    "navigationBarTextStyle": "black"
  },
  // 全局定义小程序组建所使用的样式版本
  "style": "v2",
  // 用来指明sitemap.json的位置
  "sitemapLocation": "sitemap.json"
}

    pages 数组中页面路径的第一个为项目的首页

    # project.config.json

    项目配置文件，用来记录我们对小程序开发工具所做的个性化配置

    # sitemap.json

    用来配置小程序页面是否允许微信索引 == PC 的 SEO

    disallow 将不被索引。

    注意：

    sitemap 的索引提示是默认开启的，但是如果关闭的话，需要在 project.config.json 的 setting 中配置 checkSiteMap 改为 false

    # 每个页面中的.json 配置文件

    页面中的配置项会覆盖 app.json 中 window 中的相同的配置项。

    # WXML 和 HTML 的区别

    # 1、 标签名称不同

    • HTML（div, span,img,a...）
    • WXML（view, text,image,navigator...）

    # 2、 属性节点不同

    • <a href="#">link</a><a>
    • <navigator url="/pages/list/list"></navigator>

    # 3、 提供了类似于 VUE 中的模版语法

    • 数据绑定
    • 列表渲染
    • 条件渲染

    # WXSS 和 CSS 的区别

    # 1、 新增了 rpx 尺寸单位

    • CSS 中需要手动进行像素单位换算，例如 rem
    • WXSS在底层支持新的尺寸单位rpx，在不同大小的屏幕上小程序会自动进行换算

    # 2、 提供了全局的样式和局部样式

    • 项目根目录中的 app.wxss 会作用于所有小程序页面
    • 局部页面的.wxss 样式仅对当前页面生效

    # 3、 WXSS 仅支持部分 CSS 选择器

    • 类选择器 id 选择器
    • 元素选择器
    • 并集选择器 后代选择器
    • 伪类选择器::after ::before

    # 小程序中的 js 文件

    小程序中的 js 文件分为三大类：

    # 1、 app.js

    • 整个小程序项目的入口文件，通过调用App()来启动整个小程序

    # 2、 页面的.js 文件

    • 页面的入口文件，通过调用Page()来创建并运行当前页面

    # 3、 普通的.js 文件

    • 普通的功能模块文件，用来封装公共的函数或者属性供页面使用

    # 小程序运行的宿主环境

    宿主环境即程序在运行的时候所需要的依赖环境。就比如 ios 的 APP 就只能在 ios 系统中运行，安卓系统只能支持安卓软件的运行。

    小程序的宿主环境则就是微信，可以使用微信小程序开发中所需要的 API。

    # 小程序宿主环境包含的内容

    # 1、 通信模型

    小程序中通信的主体是渲染层（WXML 和 WXSS）和逻辑层（JS）

    # 1、 渲染层和逻辑层之间的通信

    • 由微信客户端 Native进行转发

    # 2、 逻辑层和第三方服务器之间的通信

    • 由微信客户端 Native进行转发

    # 2、 运行机制

    # 小程序启动的过程

    1. 将小程序的代码包下载到本地
    2. 解析app.json全局配置文件
    3. 执行app.js小程序入口文件，调用App()创建小程序实例
    4. 渲染小程序首页
    5. 启动完成

    # 页面渲染的过程

    1. 加载解析页面的.json配置文件
    2. 加载页面的.wxml和.wxss
    3. 执行页面的.js文件，调用Page()创建页面实例
    4. 渲染完成

    # 3、 组件

    小程序中的组件是由宿主环境（微信）提供的，一共被分成 9 类：

    1、视图容器

    • view == div（块级）

    • scroll-view 可滚动的视图区域，常用来实现滚动列表效果

    • swiper&swiper-item 轮播图的容器组件和轮播图的 item 组件

    2、基础内容
    • text 文本组件 == span（行内元素） 通过 text 组件的selectable属性可以实现长按选择文本内容
    • rich-text == 富文本组件（支持把 HTML 字符串渲染成 WXML 结构） rich-text 的 nodes 属性节点可以把 HTML 字符串渲染为对应的 UI 结构
    3、表单组件

    • button 可以通过 open-type 属性调用微信提供的各种功能（客服、转发、获取用户授权、获取用户信息。。。）

    • image 默认宽度 300px 高度 240px mode 属性可以用来指定图片的裁剪和缩放模式

    • navigator 页面的导航组件 类似于 a

    4、导航组件

    5、 媒体组件

    6、 map 地图组件

    7、 canvas 画布组件

    8、 开放能力

    9、 无障碍访问

    # 4、 API

    小程序中的 API 由宿主环境微信提供。小程序官方将 API 分了三大类：

    # 事件监听 API

    以 on 开头，用来监听某些事件的触发

    # 同步 API

    以Sync结尾的 APU 都是同步 API

    同步 API 的执行结果，可以通过函数返回值直接获取，如果执行出错会抛出异常

    # 异步 API

    类似于 jQuery 中的$.ajax(options)函数，需要通过success、fail、complete接收调用的结果

    # 小程序发布上线

    开发版本提交审核->发布->线上版本

    小程序管理后台->设置->基本设置->基本信息->小程序相关信息（可用于推送）
