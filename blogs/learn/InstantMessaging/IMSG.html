    即时通信类别

    TuuOrange 4/13/2023 即时通信

    一般都是直接请求 Ajax，用到实时通信的地方较少，一般都是用 websocket 和 webworker。

    # 短轮询

    每隔一段时间向客户端发送请求，获取服务器最新的数据。但是非常消耗服务器性能，多应用于二维码扫码确认、信息通知。

    实现方式：定时器

    # Comet（长轮询、长连接）

    一种是 AJAX 的长连接（long-polling）方式，另一种是基于 iframe 和 htmlfile 的流（streaming）方式

    # 长轮询

    客户端向服务端发送 Ajax 请求，直到有新消息才返回响应信息并关闭连接，客户端处理完新消息之后再向服务器发送新的请求。

    但是保持服务器一直连接，会消耗很多性能，而且服务器一直不放开会造成堵塞，而且返回数据的顺序没有保证难管理。

    getx() {
  var that = this;
  this.$axios('x', post).then(res => {
    if(res.code === 200) {

    } else {
      getx();
    }
  })
}

    # 长连接

    在页面里嵌入一个隐藏 iframe，将隐藏的 iframe 的 src 属性设为对一个长连接的请求或是采用 xhr 请求，服务器就可以不断的给客户端发送消息。

    但是服务器也是需要一直保持连接，消耗性能。

    <iframe ref="iframe" v-show="iframeShow"></iframe>

watchIframe() {
  this.iframeWin = this.$refs.iframe.contentWindow;
  this.iframeWin.postMessage({}, "*");
  window.addEventListener("message", this.handleMessage);
  handleMessage(res) {};
}

    # SSE（Server-Sent Events）

    SSE 是一种允许服务器向客户端推送新数据的 HTML5 技术，必须借助成熟的库或框架。

    基于文本传输效率没有 websocket 高，也不是严格的双向通信，客户端发送请求给服务端不能复用之前的连接，必须要重新发起请求。

    # WebSocket

    websocket 是一个全新的、独立的协议，基于 TCP 协议，与 http 协议兼容、却不会融入 http 协议，可以在服务端和客户端建立实时的双向通信。

    浏览器为 HTTP 通信提供了 XMLHttpRequest 对象，同样的，也为 websocket 通信提供了一个通信操作接口，即 WebSocket。

    var ws = new WebSocket(wss://echo.websocket.org);
ws.onopen = function(evt) {
  ws.send("Hello WebSocket!");
}
ws.onmessage = function(evt) {
  console.log(evt.data);
}
ws.onclose = function(evt) {
  console.log('关闭连接');
}

    # Web Worker

    webworker 的作用是为浏览器创造多线程环境，允许主线程创建 worker 线程，可以多线程执行其他任务。

    但是无法访问 DOM 节点，无法访问全局变量或者全局函数，无法调用 alert 函数或者 confirm 之类的函数，无法访问 window、document 之类的浏览器全局变量。（但是，可以使用 setTimeout、setInterval 之类的函数，也可以使用 XMLHttpRequest 对象来进行 Ajax 通信）。

    适用于大数据处理和高频的用户交互。

    WebWorkers 分为：专用线程和共享线程。专用线程随当前页面的关闭而结束，共享线程可以被多个页面访问。

    # 专用线程

    使用 onmessage、post message 通信

    var worker = new Worker("worker.js");
worker.onmessage = (e) => {
  console.log(e.data);
};
worker.postMessage("主线程获取信息");

// 子线程
onmessage = (e) => {
  console.log(e.data);
};
postMessage("子线程给主线程post了一个信息");

worker.terminate(); // 在主线程中终止
self.close(); // 在子线程中终止

    # 共享线程

    SharedWorker 需要用到 port 属性，接受需要先 connet。

    let worker = new SharedWorker("sharedworker.js");
worker.port.onmessage = function (e) {};
worker.port.postMessage("data");

// 子线程
addEventListener("connect", function (event) {
  var port = event.ports[0];
  post.onmessage = function (e) {
    console.log(e.data);
  };
  // 发送
  port.postMessage("data");
  port.start();
});

worker.terminate();
self.close();

    # WebWorker 的错误监听

    worker.addEventListener(
  "error",
  function (evt) {
    alert(evt);
    worker.postMessage(10000);
  },
  false
);

    # Service Worker

    ServiceWorkers 本质上作为 web 应用程序与浏览器之间的代理服务器，也可以在网络可用时作为浏览器和网络间的代理，创建有效的离线体验，它是 webworker 的一种类型。serviceworker 可以秒开或者离线访问。

    适用于推送通知（允许用户选择从网络应用程序及时更新）。

    import { register } from 'register-service-worker';
if(process.env.NODE_ENV === 'production') {
  register('service-worker.js', {
    ready() {
      console.log('ing');
    },
    registered() {
      console.log('registered');
    },
    cached() {
      console.log('cached');
    },
    updatefound() {
      console.log('update');
    },
    updated() {
      console.log('updated');
    },
    offline() {
      console.log('offline');
    },
    error(err) {
      console.log('error', err);
    }
  })
}

    # Flash Socket

    在页面内嵌入一个使用了 Socket 类的 Flash 程序，JS 通过调用该 Flash 程序提供的 Socket 接口与服务端的 Socket 接口进行通信，JS 在收到服务端传送的信息后控制页面的显示。

    一般适用于网络游戏中，Web 端基本不适用，但是 Flash 基本已经不再使用。
